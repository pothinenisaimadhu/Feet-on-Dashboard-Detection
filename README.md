# Feet on Dashboard Detection — OOP Case 1

> Euro NCAP Out-of-Position detection: classifies each video frame as **FEET ON DASHBOARD** or **NORMAL POSTURE** for the front passenger.

---

## Pipeline

```
Input frame
  └─► PassengerDetector (YOLOv8n)        — largest person on right half → bounding box
        └─► PoseEstimator                — YOLOv8n-pose (primary) · MediaPipe (fallback)
              └─► Bootstrap (30 frames)  — auto-select standard vs high-ankle scoring mode
                    └─► DashboardROI     — normalised polygon point-in-polygon test
                          └─► score_frame — geometry: knee angle · forward_drop · elevation
                                └─► TemporalSmoother — EMA + hysteresis → stable verdict
                                      └─► Renderer → annotated output MP4
```

### Scoring logic (per frame, per leg)

1. **Negative gate first** — straight leg or low ankle_y exits immediately as NEGATIVE; ROI alone can never trigger POSITIVE.
2. **Fused geometry** — requires bent knee (`< 120°`) + forward foot (`ankle_y − hip_y ≥ 0.18`) + negative elevation.
3. **ROI as supporting evidence** — boosts score only when geometry conditions are also met.
4. **High-ankle mode** — auto-detected from first 30 frames; used when ankle_y is always elevated (e.g. profile 3 camera angle). Discriminator shifts to ROI membership + knee extension.
5. **Temporal smoothing** — EMA (α = 0.35) + hysteresis (ON = 0.55, OFF = 0.35) eliminates per-frame flicker.

## Setup

**Requirements:** Python 3.11 · pip

```bash
cd submission
pip install -r solution/requirements.txt
```

All model weights (`yolov8n.pt`, `yolov8n-pose.pt`, `pose_landmarker_lite.task`) are committed to the repo — no manual downloads needed.

## Run

### Process all videos (batch)

```bash
cd submission
python solution/run.py
```

Reads every `.mp4` under `input/` and writes annotated output to `output/` preserving the profile structure.

### Process a single video

```bash
python solution/run.py input/profile1/positive_1.mp4
```

### Streamlit UI

```bash
cd submission/solution
streamlit run app.py
```

- **Upload Video** — upload any MP4/AVI/MOV, analyse frame-by-frame, optionally export annotated video
- **Webcam** — live detection (local only; not available on Streamlit Cloud)

## Output Video Annotations

Each output frame contains:

| Element | Description |
|---|---|
| Orange polygon | Dashboard ROI region |
| Amber bounding box | YOLO passenger selection + confidence |
| Large circles | Ankle joints — red if inside ROI, yellow otherwise |
| Small circles | Knee joints (cyan) |
| Leg line | Knee → ankle segment |
| Top-left label | **FEET ON DASHBOARD** (red) or **NORMAL POSTURE** (green) |
| Debug line | Knee angle, forward_drop, elevation, ROI flag, pose backend |
| Confidence | EMA-smoothed score 0.0 – 1.0 |

## Configuration

All parameters are in `solution/config.py`:

| Parameter | Value | Effect |
|---|---|---|
| `IMGSZ` | 320 | YOLO inference resolution |
| `DETECT_EVERY` | 4 | Reuse passenger box every N frames |
| `DETECT_MISS_MAX` | 12 | Force re-detect after N missed frames |
| `PASSENGER_SIDE` | `"right"` | Passenger is right half of frame |
| `PASSENGER_HIP_X_MIN` | 0.55 | Hip x-position filter |
| `KP_MIN_CONF` | 0.20 | Minimum keypoint confidence |
| `KNEE_BENT_THRESHOLD` | 120° | Bent-leg detection |
| `FORWARD_DROP_MIN` | 0.18 | `ankle_y − hip_y` for forward foot |
| `DASHBOARD_ROI_NORM` | lower-right quad | Dashboard region (normalised) |
| `EMA_ALPHA` | 0.35 | Temporal smoothing factor |
| `HYST_ON` / `HYST_OFF` | 0.55 / 0.35 | Hysteresis thresholds |

## Limitations

- Dashboard ROI polygon is hand-tuned for the Hyundai Santa Fe + this specific camera mount. A different vehicle requires re-tuning `DASHBOARD_ROI_NORM`.
- Occluded feet (blanket, bag) cause false negatives — a dedicated foot detector would close this gap.
- No depth information used; the provided camera intrinsics are not applied (future improvement).
- Single-passenger assumption — one passenger box is selected per frame.
